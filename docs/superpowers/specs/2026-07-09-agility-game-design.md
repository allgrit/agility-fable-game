# Agility Trial! — дизайн-документ

Дата: 2026-07-09. Статус: утверждён (автономный режим).

## Концепция

Аркадная игра про собачий спорт аджилити. Игрок — **собака** на соревновательной трассе.
Хендлер бежит рядом и выкрикивает команды; задача игрока — **правильно и вовремя реагировать
на команды** (QTE). Точность реакции определяет скорость, чистоту прогона и итоговый результат.

Платформа: браузер, HTML5 Canvas 2D + WebAudio (синтезированный звук), ванильный JS, ноль зависимостей.

## Геймплей

### Цикл
Меню (выбор породы) → Забег по трассе → Экран результатов → Следующий класс (Novice → Open → Excellent → Masters) с ростом сложности.

### Движение
Собака бежит автоматически по сплайну Catmull-Rom через снаряды. Игрок не рулит — он реагирует
на команды. Скорость меняется от качества QTE (Perfect → ускорение + комбо, Miss → фолт и потеря времени).

### QTE по снарядам (аутентичные команды)
| Снаряд | Команда хендлера | Ввод | Провал |
|---|---|---|---|
| Барьер / шина / стена / длинный прыжок | «Хоп!» | Space в тайминг-окне | сбитая планка (5 faults) |
| Туннель | «Туннель!» | ↓ | отказ (refusal, круг, потеря времени) |
| Слалом (12 стоек) | «Змейка!» | ←→←→ ритмично, 6 тактов | пропуск стойки (5 faults) |
| Горка / бум (contact) | «Вперёд!» → «Зона!» | удерживать ↑, отпустить в жёлтой контактной зоне | пропуск зоны (5 faults) |
| Качели | «Качели!» | ↑, затем Space в момент опускания доски | 5 faults |
| Стол | «Стол! Ждать!» | удерживать Space 3 с (шкала) | ранний сход (5 faults) |

Градации тайминга: **Perfect / Good / Late / Miss**. Perfect растит комбо (x2..x8) и даёт burst скорости.

### Судейство
- SCT (Standard Course Time) рассчитывается из длины трассы и класса.
- Faults: 5 за ошибку + time faults за превышение SCT (1/с).
- Clean Run (0 faults) → титул «Q» + конфетти. Итог: 0–3 звезды + рейтинг.

### Породы (статы)
- **Бордер-колли** — скорость высокая, окно QTE узкое.
- **Шелти** — скорость средняя, окно QTE шире (точность).
- **Джек-рассел** — скорость ниже, комбо растёт быстрее.

## Процедурная генерация трасс

Генератор (seed-детерминированный) размещает 12–18 снарядов на поле ~52×36 м:
- обязательны: слалом ×1, контактный снаряд ≥1, туннель ≥1, серии барьеров;
- лимиты по правилам: шина ≤1, слалом ≤1, стол ≤1;
- паттерны: serpentine (S-зигзаг), pinwheel (колесо барьеров);
- межснарядная дистанция 5–10 м, углы поворота ограничены, самопересечения пути штрафуются при генерации (retry);
- путь — Catmull-Rom через точки входа/выхода снарядов; финишная арка.

## Презентация («полировка»)

- Псевдо-изометрия: эллиптические тени, вертикальный масштаб, трава с шумом, ринг с ограждением, зрители, судья.
- Анимации: процедурная собака (бег с фазами ног, прыжок с дугой, уши/хвост по физике), хендлер бежит параллельным курсом, речевые пузыри команд.
- Камера: слежение с lookahead, плавный zoom, screen shake на ошибках, hit-stop на Perfect.
- Частицы: пыль из-под лап, планки разлетаются, конфетти на Q, искры Perfect.
- SFX (WebAudio-синтез): свисток старта, топот, «вжух» прыжка, стук планки, ритм-клики слалома, скрип качелей, лай, гул и аплодисменты толпы, фанфары финиша.
- HUD: номер снаряда n/N, таймер против SCT, faults, комбо, полоса скорости.

## Архитектура

```
index.html          — каркас, canvas, подключение скриптов
css/style.css       — оформление страницы/HUD-оверлеев
js/rng.js           — seeded RNG (mulberry32)
js/spline.js        — Catmull-Rom, длины дуг, выборка по дистанции
js/course.js        — генератор трасс (чистая логика, тестируемая)
js/qte.js           — конечный автомат QTE (чистая логика, тестируемая)
js/scoring.js       — SCT, faults, звёзды, титулы (чистая логика, тестируемая)
js/audio.js         — WebAudio-синтезатор SFX и толпы
js/particles.js     — система частиц
js/render.js        — отрисовка мира, снарядов, собаки, хендлера, толпы
js/game.js          — состояние забега: собака на сплайне, привязка QTE, камера
js/main.js          — state machine (menu/run/results), input, loop
tests/logic.test.mjs— node-тесты чистой логики
tools/screenshot.mjs— Playwright-харнесс скриншотов
```

Чистая логика (course/qte/scoring/spline/rng) не трогает DOM — грузится и в браузере, и в node-тестах.

## Тестирование
1. Node-тесты: генератор (валидность трасс на 200 сидах), QTE-тайминги, скоринг.
2. Playwright-харнесс: скриншоты меню/забега/результатов, прогон с эмуляцией ввода, проверка консоли на ошибки.
3. Адверсарное ревью воркфлоу-агентами (bugs / gameplay / polish) + фиксы.
